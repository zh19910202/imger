解决 "Chrome runtime API不可用" 问题

🎯 问题描述
当您看到 "Chrome runtime API不可用" 错误时，这通常表示：
  • Chrome扩展未正确加载到浏览器中
  • 扩展权限配置有问题
  • 在错误的网站或环境中运行
  • 扩展文件损坏或不完整
🚀 快速解决方案

📋 检查清单（按顺序执行）

  • 确认使用Chrome浏览器（版本88+）
  • 打开扩展管理页面：chrome://extensions/
  • 开启"开发者模式"开关
  • 检查AnnotateFlow Assistant扩展是否存在
  • 确认扩展状态为"已启用"
  • 点击扩展的"重新加载"按钮
  • 在腾讯QLabel网站测试功能
🔄 重新安装扩展
如果上述步骤无效，请重新安装扩展：
  1. 在扩展管理页面移除现有扩展
  2. 点击"加载已解压的扩展程序"
  3. 选择项目文件夹：d:\app\imgerV2
  4. 确认安装成功
⚠️ 注意：确保项目文件夹包含所有必要文件：
manifest.json, background.js, content.js, popup.html, popup.js
🌐 网站兼容性检查
AnnotateFlow Assistant主要为以下网站设计：
https://qlabel.tencent.com/*
✅ 推荐测试环境：
  • 访问：https://qlabel.tencent.com/
  • 登录您的账户
  • 进入标注任务页面
  • 按D键测试下载功能
🔍 高级故障排除
如果问题仍然存在：
  1. 检查Service Worker：
    • 在扩展管理页面找到扩展
    • 点击"检查视图 service worker"
    • 查看控制台是否有错误
  2. 清除浏览器缓存：
    • 按Ctrl+Shift+Delete
    • 选择"全部时间"
    • 清除缓存和Cookie
  3. 重启Chrome浏览器
  4. 检查Chrome版本是否为最新
🆘 仍然无法解决？
请运行诊断工具获取详细信息，或检查以下文件是否存在：
d:\app\imgerV2\manifest.json d:\app\imgerV2\background.js d:\app\imgerV2\content.js d:\app\imgerV2\popup.html d:\app\imgerV2\popup.js
